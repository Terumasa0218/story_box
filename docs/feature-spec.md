# Feature Spec

## 1. 物カード

ユーザーが1つの品を1枚のカードとして登録する。

Fields:

- 写真
- 品名
- カテゴリ
- 保管場所
- 持ち主
- いつ頃の物か
- 思い出メモ
- 公開範囲
- 行き先ステータス

Categories:

- 写真・アルバム
- 食器
- 着物・服
- アクセサリー
- 趣味の道具
- 手紙・書類
- 家具
- 子どもの作品
- 記念品
- 家電
- よく分からない物

## 2. 思い出メモ

長文を書かせず、短い質問で記録を作る。

Prompt examples:

- これは誰から来たものですか？
- いつ頃から持っていますか？
- これを見ると誰を思い出しますか？
- なぜ捨てづらいですか？
- 誰かに譲るなら、誰が喜びそうですか？
- 手放すなら、どんな形で残したいですか？

## 3. 音声メモ

30秒ほど話すだけで、AIが短い思い出文に整える。

Output styles:

- そのまま記録
- 家族に見せる文
- ありがとうカード用の短文
- SNS共有用の匿名化文

## 4. 行き先ステータス

物の今後をゆるく整理する。

Statuses:

- 残す
- 家族に聞く
- 譲る
- 売る
- 寄付する
- 写真だけ残して手放す
- 処分する
- 保留

## 5. 家族共有

LINEなどで共有できる公開ページを生成する。

家族側はアプリ登録なしで回答できる。

Responses:

- 欲しい
- いらない
- 写真だけ見たい
- 相談したい

Design notes:

- 最初は期限管理を入れない
- 複雑な公平分配や抽選も入れない
- 「欲しい人がいたか」を残すだけにする
- 思い出メモ、価格、書類情報は共有範囲を選べるようにする

## 6. 処分ガイド

品名とカテゴリから、処分の方向性だけを示す。

Guide labels:

- 自治体確認が必要
- 粗大ごみ候補
- 小型家電回収候補
- リサイクルショップ向き
- フリマ向き
- 寄付できる可能性あり
- 専門業者に相談向き
- 書類・個人情報に注意

地域差があるため、処分方法を断定しない。市区町村名を入れた場合は、自治体サイトや検索リンクへ誘導する。

## 7. ありがとうカード

手放した品を、写真と一言メモで残す。

Use cases:

- 自分だけで見返す
- 家族に共有する
- SNSに匿名化して投稿する
- PDFや冊子にまとめる

Safety:

- 個人名
- 住所
- 学校名
- 会社名
- 顔写真
- 連絡先
- 病歴や金銭トラブル
- 誹謗中傷につながる表現

上記は共有前に検出・マスク・警告する。

## MVP User Flow

1. 品物を撮影する
2. 品名とカテゴリを入れる
3. 質問に1つだけ答える、または音声で話す
4. 行き先ステータスを選ぶ
5. 必要ならLINEで家族に共有する
6. 家族の回答を見て、残す・譲る・手放すを決める
7. 手放した品はありがとうカードとして残る

## Non-Goals for MVP

- 相続、遺言、法務アドバイス
- 本格的な資産評価
- AI鑑定、査定
- フリマ出品代行
- 家系図
- 大容量写真ストレージ
- 断捨離SNSタイムライン
- 複雑な家族間分配ロジック
